# MCS-217 Checklist

MCS-217 is a software engineering lab sequence. The Railway Reservation System should remain the running case study for sessions 1-14, then the later sessions should handle usage scenarios, structured programming review, reverse engineering, and Library Information System work.

| Session | Topic | Status | Expected Deliverable |
| ------- | ----- | ------ | -------------------- |
| 1 | Scope statement for RRS | Draft | Scope statement with assumptions and boundaries. |
| 2 | Effort and cost estimation | Draft | Estimation method, assumptions, calculations, final estimate. |
| 3 | SRS for RRS | Skeleton added | Complete SRS document. |
| 4 | DFD, ERD, Data Dictionary | Skeleton added | Levelled DFDs, ERD, relationship notes, dictionary. |
| 5 | Modular design for RRS | Skeleton added | Module decomposition and interface summary. |
| 6 | User interface design for RRS | Skeleton added | Screens, navigation flow, validation rules. |
| 7 | Matrix multiplication in C using pointers | Skeleton added | C program, output, explanation. |
| 8 | Test cases for Session 7 | Skeleton added | Unit, module, and integration test cases. |
| 9 | Matrix transpose web page | Skeleton added | HTML/JS page and output. |
| 10 | Test cases and report for Session 9 | Skeleton added | Test cases and test report. |
| 11 | Correct but poor-quality program | Skeleton added | Program and quality justification. |
| 12 | Correct but unreliable program | Skeleton added | Program and reliability justification. |
| 13 | Develop RRS | Skeleton added | Integrated software based on earlier specs. |
| 14 | Change control for RRS | Skeleton added | Change list, impact analysis, implementation report. |
| 15 | Usage scenarios | Skeleton added | Scenario set for selected software. |
| 16 | Structured programming review | Skeleton added | Code review and rewritten structured version. |
| 17 | Recreate external application portion | Skeleton added | Application analysis and implementation attempt. |
| 18 | Continue external application recreation | Skeleton added | Improved implementation and notes. |
| 19 | Library Information System requirements | Skeleton added | Requirements gathered from library visit/user. |
| 20 | Library Information System implementation | Skeleton added | LIS software and validation report. |
