## Problem Statement

Suppose that you need to build software for a Railway Reservation System. Write a statement of scope that describes the software.

## Statement of Scope

The Railway Reservation System is used to Create Reservation, Cancel Reservation, Viewing/updating Train Information, viewing reservation status, generating reports, Updating/Viewing train schedule, updating reservation details, Managing journey time, Arrival/Departure time, Seat availability etc. online. This System will reduce the time and effort for going to railway office and applying for these facilities for transportation.
 
Reservation is created from any Railway Station which is updated automatically in all the stations. Hence, there is no confusion to the Reservation Clerk in all the stations to create the Reservation. This can be possible by maintaining Global Database. Representatives present at different stations can access the global database and can easily understand the remaining reservation seats. It provides the ability to create reservation from different places for a train and the clients can easily be able to book reservation seats. It provides various options and diversity to the customer as well as more time to book tickets:

- The Seats of Reservation cannot be more than the seats of Train at that date
- Sending the reservation details to the registered email and mobile number such as confirmation message or Any changes in their booking and railway timing should be notified by message to the user.
- All payment modes is supported. Easy and fast refund on cancelation of ticket.
- Booking railway staff to help old age and physically disabled person.
- Google map linked for trekking route, live location sharing facilities and time and other facilities related to health and meal.
- Twenty-four-7 accessibility and response service.
- Language support: it should be multilingual.
- Voice recognition system also be available which reduces the effort and time to write each detail manually. Lost and found section also available in the system.
- The software will run on both mobile and desktop and on every browser.
- User friendly, simple and easy interface for users. Proper guidance for using the software should be there.
- Secure access of private information (client's subtleties). Confidential data such as account or card details are not stored anywhere and is fully secured system.

The system is so secured and clerk and manager utilize it. Nobody can able to access the system without his or her permission because of providing login facility to the system. The password is in the form of cipher text by using cryptography technology, so it cannot be hacked by any person. The global database can pass through network in order to utilized by managers or clerks at different places. So, we want to provide network security because of the data not hacked by the other persons when it is going through network cables. This network security is provided by the cryptography technology.
